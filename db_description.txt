Назвал базу данных vkinder_db
Логин и пароль запрашиваю через input()
Помимо обговоренных таблиц добавил в нее таблицу для черного списка
При каждом запуске таблицы стираются и создаются заново

Доступные функции:
add_user_to_db(user_id=user_id, name=name, surname=surname, gender=gender, age=age, city=city)
Добавляет юзера в базу данных.
Аргументы: session - текущая сессия, id юзера, его имя, его фамилия, его пол, его возраст, его город
Возвращает True, если юзер добавлен в базу данных и False в противном случае

add_favorite_to_db(user_id, favorite_id):
Для юзера-пользователя добавляет в избранное просмотренного юзера
Аргументы: session - текущая сессия, user_id - id юзера, которого нужно добавить в избранное, favorite_for_id - id юзера-пользователя, для которого составляем избранное
Возвращает True, если юзер добавлен в базу данных и False в противном случае

add_photo_to_db(session, photo_list, favorite_id)
Аргументы: session - текущая сессия, photo_list - список id-шников фотографий, favorite_id - id юзера из ибранного, чьи фотки добавляются в базу
Думаю, эту функцию нужно применять совместно с add_favorite_to_db
Возвращает True, если все фотки добавлены в базу данных и False в противном случае

get_favorites(session, user_id)
Возвращает список избранных пользователей для юзер-пользователя
Аргументы: session - текущая сессия, user_id - id юзера, для которого запрашиваем избранное
Возвращает список int значений id-шников юзеров из избранного, либо False, если произошла ошибка

add_user_to_blacklist(session, user_id, blocked_for_id):
Для юзера-пользователя добавляет в черный список просмотренного юзера
Аргументы: session - текущая сессия, user_id - id юзера, которого нужно добавить в черный список, blocked_for_id - id юзера-пользователя, для которого составляем черный список
Возвращает True, если юзер добавлен в ЧС и False в противном случае

get_blacklist(session, user_id)
По аналогии с get_favorites(), возвращает список id-шников из черного списка
